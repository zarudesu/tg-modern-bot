# 🤖 Modern Telegram Bot

Современный, масштабируемый Telegram бот для IT-администрирования с модульной архитектурой, системой ролей и интеграциями.

## ✨ Особенности

- 🏗️ **Модульная архитектура** - легко добавлять новый функционал
- 🔐 **Admin-only доступ** - бот доступен только администраторам
- 📊 **Полное логирование** - все действия записываются в БД
- 🐳 **Docker-ready** - готов к продакшн развертыванию
- ⚡ **Async/await** - полностью асинхронный код с высокой производительностью
- 🔒 **Type hints** - полная типизация для надежности кода
- 🔄 **Graceful shutdown** - корректное завершение работы всех компонентов
- 📈 **Мониторинг** - health checks и метрики производительности

## 🎯 Текущий функционал

### ✅ Базовая функциональность
- Система аутентификации только для админов
- Полное логирование всех действий пользователей
- База данных PostgreSQL с автоматическими миграциями
- Кэширование сессий через Redis
- Базовые команды (/start, /help, /profile, /ping)

### ✅ Модуль журнала работ **[ЗАВЕРШЕН]**
- **Интерактивное заполнение** записей о выполненных работах
- **Пошаговый интерфейс** с inline-клавиатурами
- **Множественный выбор исполнителей** (JSON: ["Тимофей", "Дима"])
- **Отслеживание создателя записи** для всех записей
- **Система упоминаний исполнителей** в Telegram
- **Настройки уведомлений** пользователей (ЛС + упоминания в чате)
- **Готовая структура для n8n webhook** с полными данными
- **Команды**: `/journal`, `/history`, `/report`
- **База данных PostgreSQL** с оптимизированными индексами

### 🔗 Готово к интеграции
- **n8n workflow** - настройка получения webhook от бота
- **Google Sheets подключение** - автоматическая отправка записей
- **Переменные окружения**: N8N_WEBHOOK_URL, GOOGLE_SHEETS_ID

## 🚀 Быстрый старт

### 1. Подготовка проекта
```bash
# Переход в директорию проекта
cd /Users/zardes/Projects/tg-mordern-bot

# Копирование переменных окружения
cp .env.example .env
# ⚠️ Обязательно отредактируйте .env со своими настройками!
```

### 2. Настройка .env файла
```bash
# Telegram Bot Configuration
TELEGRAM_TOKEN=your_bot_token_here
TELEGRAM_API_ID=your_api_id
TELEGRAM_API_HASH=your_api_hash
ADMIN_USER_IDS=123456789,987654321  # ID админов через запятую

# Database Configuration
DATABASE_URL=postgresql+asyncpg://bot_user:bot_password@localhost:5432/telegram_bot
REDIS_URL=redis://:redis_password@localhost:6379/0
```

### 3. Запуск в режиме разработки
```bash
# Гибридный режим (рекомендуется для разработки)
make dev          # БД в Docker, бот локально

# Или полный Docker стек
make full-up      # Всё в Docker контейнерах
```

### 4. Проверка работоспособности
```bash
# Проверка статуса сервисов
make status

# Просмотр логов
make logs-bot

# Отправьте /start своему боту в Telegram
```

## 🔐 Безопасность

### Аутентификация и авторизация:
- **Только админы** могут использовать бота
- **AuthMiddleware** проверяет права на каждый запрос
- **Автоматическая блокировка** неавторизованных пользователей
- **Полное логирование** всех действий

### Обработка данных:
- **Валидация** всех входных данных через Pydantic
- **SQL инъекции** защита через SQLAlchemy ORM
- **Rate limiting** для защиты от спама
- **Graceful error handling** без утечки данных

## 📊 Мониторинг и логирование

### Типы логов:
- **User actions** - действия пользователей
- **API calls** - вызовы внешних API
- **Performance metrics** - метрики производительности
- **Error tracking** - отслеживание ошибок

### Метрики:
- Время выполнения запросов
- Количество активных пользователей
- Статистика использования команд
